# v1.0.0

Release date: June 2026

## Highlights

- Establishes the 1.0.0 baseline for the Customs Calculator web platform.
- Aligns release metadata across package, changelog index, and README current-release documentation.
- Adds release consistency guardrails to reduce version drift risk in future releases.

## Platform Baseline

v1.0.0 rolls up the production feature set delivered through the v0.x cycle, including:

- Core landed-cost calculations (duty, VAT, surcharge-aware taxable base, excise, trade remedies).
- Section 800 exemptions, valuation-risk indicators, and penalty estimation in calculation outputs.
- Multi-currency flow with PHP-based computation pipeline and official-rate-aware conversion fallback behavior.
- HS search quality improvements, chapter drill-down UX, and official lookup assistance with cache fallback.
- Admin governance surfaces for imports, review queues, and runtime/source management.

## Release Process Hardening

To make release operations safer and repeatable, v1.0.0 introduces:

- `npm run release:check` script to validate release metadata consistency.
- `npm run release:prepare` pre-release quality gate helper.
- A CI workflow that runs release consistency checks on push/PR when release files are touched.

The consistency checker validates alignment between:

- `package.json` version
- `package-lock.json` top-level version fields
- `CHANGELOG.md` current release entry and package version note
- `README.md` current release heading

## Notes

- This release does not introduce a breaking runtime API change by itself; it marks a stable, maintained baseline and release-governance milestone.
- Future release notes should continue to be added in `docs/changelog/vX.Y.Z.md` and indexed from `CHANGELOG.md`.
