# v0.5.0

Release date: April 30, 2026

Last updated: May 2, 2026 (post-release hardening pass)

## Highlights

- Added full PHL 2026 customs compliance fee and tax engine, covering all CMTA/CAO administrative fees, excise tax categories, corrected VAT base, BOC weekly FX rates, and all three entry tiers (de minimis / informal / formal).
- Added goods classification engine for Philippine import regulation: CMTA import type (Free / Regulated / Restricted / Prohibited), agency clearance requirements, strategic trade (STMO/RA 10697) flag, VAT-exempt goods (NIRC Sec. 109) flag, and FTA Certificate of Origin form identification.
- Import Classification panel added to the calculator results display, showing import type badge, required agency clearances, CoO alert, strategic trade warning, and VAT-exempt note.
- CoO hint shown in the calculator form when a non-MFN FTA schedule is selected.
- Post-release hardening pass added reliability and UX improvements across lookup, ingestion, calculation outputs, mobile UX, and admin audit visibility.
- Test suite now at 197 tests; all passing.

## Post-Release Hardening (May 2, 2026)

### Backend and Data Reliability

- **Official HS lookup resilience (`OfficialHsLookupService`)**
	- Added in-flight request deduplication for repeated query bursts.
	- Added stale-cache fallback when upstream Tariff Commission fetch fails after TTL expiry.
	- Cache behavior improved with touch-on-hit LRU ordering.

- **Tariff ingestion fallback (`TariffDataIngestionService`)**
	- `parseHtmlTables` now falls back to text-pattern extraction when structured HTML table parsing yields zero rows.
	- Fallback rows are imported with lower confidence for review-first governance.

- **Currency valuation source precedence (`CurrencyConverter`)**
	- PHP valuation pairs now prioritize official-rate chain: **BSP -> BOC -> live market -> cache -> fallback**.

- **Compliance messaging refinements (`ComplianceChecker`)**
	- Added explicit de minimis warning that exemptions do not waive regulated-goods documentary clearances.
	- Reinforced FDA/NTC clearance guidance in requirement/restriction outputs.

### Calculation and Reporting Enhancements

- **Extended assessment outputs**
	- Added trade remedy duty outputs (anti-dumping, countervailing, safeguard, total).
	- Added surcharge/penalty outputs (undervaluation surcharge, misclassification surcharge with clerical bypass, late-payment interest).
	- Added final `totalPayable` output in addition to `totalLandedCost`.

- **Official PDF-style report improvements (`DocumentGenerator`)**
	- Added lines/sections for trade remedies, penalties, and total payable aligned with informal-entry style presentation.

## Goods Classification Engine

New `importClassification` service classifies any 10-digit AHTN/HS code against Philippine regulatory frameworks:

- **Import type (CMTA Sec. 116â€“118):** `free` | `regulated` | `restricted` | `prohibited` â€” chapter-level defaults for all 99 HS chapters with heading-level overrides for 30+ sensitive headings.
- **Agency clearance:** returns abbreviated agency codes (e.g., FDA, FPA, NTC, SRA) and full agency names for all clearance requirements per heading.
- **Strategic trade flag:** identifies goods under RA 10697 (Strategic Trade Management Act); covers radioactive materials (2844), radar/navigation (8526), optical/laser equipment (9013), explosives (3601â€“3604), and other controlled items. `isStrategicTradeGood` + `strategicTradeNotes` exposed on result.
- **VAT-exempt flag:** identifies goods exempt under NIRC Sec. 109 including agricultural/marine staples, medicines, fertilizers, seeds, and books. `isVatExempt` + `vatExemptBasis` exposed on result.
- **FTA Certificate of Origin:** `requiresCertificateOfOrigin` set for non-MFN FTA schedule codes (ATIGA, ACFTA, AHKFTA, AJCEPA/JPEPA, AIFTA, AANZFTA, RCEP, ACDFTA, PH-EFTA). Form identifiers returned: Form D, Form E, JPEPA CoO, RCEP CoO, Form AI, Form AANZ, RCEP CoO, Form PH-EFTA.
- **Warnings:** EO 156 vehicle import restrictions (8703), Dangerous Drugs Board (2939/3002), CITES (0106/4101/0407), strategic goods advisory â€” surfaced as a `warnings[]` array on the result.
