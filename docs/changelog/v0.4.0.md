# v0.4.0

Release date: April 28, 2026

## Highlights

- Added conflict-aware tariff review workflow with side-by-side decisioning support (`Use Incoming` vs `Keep Existing`).
- Added provenance drill-down for review rows, including source context, import job context, and recent rate-change audit history.
- Added authority-ranked live HS search merge behavior (official live > official cache > local catalog), with deterministic dedupe/ranking.
- Added explicit tariff history browsing mode and history API endpoint for timeline-style catalog inspection.
- Added staged cutover runtime controls:
  - `stagedCutoverEnabled`
  - `cutoverCoverageThreshold`
  - `fullSyncIdempotencyGuardEnabled`
- Added full-sync idempotency regression coverage and improved import idempotency handling.
- Fixed `uq_tariff_rates_version` collision behavior:
  - duplicate version rows are now handled idempotently
  - same-key, changed-value rows are updated safely instead of hard failing
  - seed insert guard now aligns with version uniqueness rules
- Bumped package version to `0.4.0`.

## Upgrade Notes (Operators)

- Review runtime settings after upgrade:
  - `stagedCutoverEnabled` controls whether cutover recommendation can become `true`.
  - `cutoverCoverageThreshold` defines the MFN coverage target used for cutover readiness.
  - `fullSyncIdempotencyGuardEnabled` prevents reprocessing duplicate full-sync source references.
- In Admin review queues, conflict rows now require explicit decisioning (`Use Incoming` or `Keep Existing`) and include provenance drill-down details.
- In Tariff Browser, use `Historical Timeline` mode to inspect prior rate versions and effective/end-date windows.
- If integration scripts parse live-search responses, account for ranked authority metadata (`authorityRank`, `authorityLabel`) in result rows.
