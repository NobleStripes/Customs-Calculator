# v0.6.0

Release date: May 2026

## Highlights

- Frontend UX polish pass across all pages and components (Phase 1A): accessible error messaging, reset flow, required-field indicators, sticky results bar, fee acronym tooltips, and export feedback.
- Admin review queue pagination and confidence-level chips (Phase 2).
- DB-managed regulatory seed URL table: admin UI for full CRUD over the URLs that the auto-fetcher uses to discover tariff and BOC memoranda updates (Phase 3).
- Test suite expanded: regulatory_sources CRUD coverage and MFN fallback behavior verified.

## Frontend UX Polish (Phase 1A)

### Calculator (`Calculator.tsx` / `Calculator.css`)

- Added `ref`-based scroll-into-view on error: when calculation fails, the error message is automatically scrolled into view.
- Error div now carries `role="alert"` and `aria-live="assertive"` for screen-reader accessibility.
- Required-field asterisks added to HS Code and FOB Value labels (`<span class="required-asterisk" aria-hidden="true">*</span>`; styled red).
- Excise banner displayed above the excise section when an excise-taxable category is detected.
- **Reset button** added alongside the Calculate button (`.calculator-form-actions` flex container). Reset clears all form fields, detected excise category, results, errors, and HS lookup selection.

### Calculation Results (`CalculationResults.tsx` / `CalculationResults.css`)

- **Sticky total bar** at the top of the results panel shows Total Landed Cost and, when penalties exist, the Total Payable (incl. penalties). Implemented as `.results-sticky-bar`.
- Export-success feedback auto-hides after 5 seconds using a `useRef` timer (clears previous timer on repeat exports).
- **Abbreviation tooltips** via `<abbr title="...">` on all fee codes: CUD, TC, IPF, CSF, CDS, IRS, LRF — styled with dotted underline and `cursor: help`.

### Settings (`Settings.tsx` / `Settings.css`)

- Unsaved-changes badge (amber pill) shown in the Settings heading when local state differs from persisted settings.
- Save button disabled when there are no unsaved changes.
- Reset now prompts for confirmation via `window.confirm` before discarding local edits.

### HS Code Search (`HSCodeSearch.tsx` / `HSCodeSearch.css`)

- "Searching…" slow-search feedback shown after 2 seconds of an in-progress search (amber fade animation via `.search-slow`).

### Tariff Browser (`TariffBrowser.tsx` / `TariffBrowser.css`)

- Sort buttons (`.sort-btn`) on table columns.
- Row count indicator (`.tariff-row-count`).
- Screen-reader-only class `.sr-only` added for accessible labels.

### Sidebar (`Sidebar.tsx` / `Sidebar.css`) and App (`App.tsx` / `App.css`)

- Navigation and layout UX refinements.

### Batch Import (`BatchImport.tsx` / `BatchImport.css`)

- UX polish pass on import flow.

## Admin Review Queue Improvements (Phase 2)

- **Pagination** on the Review Queue tab: `reviewPage` state, `pagedReviewRows` useMemo, page navigation controls.
- **Confidence chips** on each review row: color-coded badges (`.confidence-chip`, `.confidence-high`, `.confidence-mid`, `.confidence-low`) for high / medium / low confidence classification.
- Page resets to 0 when a new job is selected (`handleSelectJob` calls `setReviewPage(0)`).

## DB-Managed Regulatory Seed URLs (Phase 3)

### Backend

- **New `regulatory_sources` table** in SQLite schema (`database.ts`): columns `id`, `source_key`, `url`, `label`, `is_active`, `notes`, `created_at`, `updated_at`. Migration uses `PRAGMA table_info` + `ALTER TABLE` (no drop-and-recreate).
- **`RegulatorySourceRow` type** exported from `database.ts`.
- **CRUD functions** exported: `getRegulatorySources`, `createRegulatorySource`, `updateRegulatorySource`, `deleteRegulatorySource`.
- `seedRegulatorySources` called in `seedInitialData` — seeds the canonical BOC and Tariff Commission URLs from `REGULATORY_SEED_URLS` constant if the table is empty.
- `websiteFetcher.ts` now looks up seed URLs from the `regulatory_sources` table with a 60-second in-memory cache, falling back to the static `REGULATORY_SEED_URLS` constant on DB error. `invalidateRegulatorySourcesCache` exported for use after mutations.
- **4 new protected API endpoints** in `server/index.ts` (all require `X-Admin-Api-Key`):
  - `GET /api/admin/regulatory-sources`
  - `POST /api/admin/regulatory-sources`
  - `PUT /api/admin/regulatory-sources/:id`
  - `DELETE /api/admin/regulatory-sources/:id`

### Frontend

- `RegulatorySourceRow` type declared in `appApi.ts`.
- `/api/admin/` path prefix added to `isProtectedAdminPath()` so the admin key header is injected automatically.
- 4 CRUD methods added to `appApi`: `getRegulatorySources`, `createRegulatorySource`, `updateRegulatorySource`, `deleteRegulatorySource`.
- **New "Regulatory Sources" tab** in the Admin panel:
  - Table listing all sources with Source Key, URL (clickable), Label, Active toggle, Delete button.
  - Add form with source_key `<select>` (boc / bir / tariff-commission / coa / customs-modernization), URL input, optional label and notes fields.
  - Success/error feedback inline.

## Tests (Phase 4)

- **`regulatory_sources` CRUD**: 4 new tests in `tariffCalculator.test.ts` covering create-and-retrieve, `is_active` toggle, delete, and update of non-existent ID.
- **MFN fallback**: 2 new tests verifying that `calculateDuty` falls back to MFN when the requested schedule has no approved row, and that a valid result is returned for MFN.
- Pre-existing test corrected: `'throws a handled error when a selected tariff schedule has no approved row'` updated to reflect actual MFN-fallback behavior.
- Package version bumped to `0.6.0`.
